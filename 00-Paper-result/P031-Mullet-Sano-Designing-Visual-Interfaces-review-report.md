## 二、文献定位

该文是对 Mullet 与 Sano *Designing Visual Interfaces* 的书评。它对《现当代设计史》的价值在于说明 1990 年代软件界面设计如何开始吸收平面设计、视觉传达、格式塔、符号学和版式原则，反映了图形设计知识进入软件开发和 HCI 领域的历史过程。

## 三、核心内容摘要

书评指出，软件工程共同体长期不充分理解图形设计专业，常把图形设计视为为软件“美化外观”的艺术性工作，而忽视视觉原则在界面设计中的重要性。Mullet 与 Sano 的书通过大量界面截图，将传统印刷和平面设计原则应用于 GUI，包括统一、对比、和谐、分组、视觉层级、平衡、网格、抽象图像、图标与风格一致性等。书评肯定该书从视觉传播角度解释界面设计，但也指出其主要关注静态对话框、图标和窗口布局，较少处理导航、动态交互和设计流程。

## 四、研究对象

- 图形用户界面；
- 软件中的视觉传达；
- 平面设计原则向 GUI 的迁移；
- 图标、窗口、网格、视觉层级；
- HCI 中设计概念的不同理解。

## 六、核心概念

| 概念 | 英文 | 教材解释方向 |
|---|---|---|
| 视觉界面设计 | visual interface design | 运用视觉传达原则组织软件界面的图像、文字、层级和布局。 |
| 图形用户界面 | graphical user interface | 通过窗口、图标、菜单、指针等图形元素支持人机交互的界面。 |
| 视觉层级 | visual hierarchy | 通过大小、位置、对比和组织方式表达信息重要性。 |
| 网格 | grid | 保持不同界面屏幕一致性和秩序的布局系统。 |
| 图标抽象 | icon abstraction | 将现实对象提炼成适合屏幕和文化语境识别的图形符号。 |

## 七、论证路径

书评先指出软件界面对图形设计知识的低估，再说明 Mullet 与 Sano 如何从传统印刷与视觉设计中提取原则并应用到 GUI。随后概述书中统一、对比、分组、层级、网格、图像抽象和风格系统等技术，并批评其缺少动态交互和设计流程讨论。最后，书评肯定该书作为 1980—1990 年代屏幕图形历史参考的价值。

## 九、教学转化建议

- 课堂问题：界面设计是编程问题、视觉传达问题，还是交互问题？
- 讨论题：为什么传统平面设计原则仍然适用于软件界面？
- 作业建议：分析一个旧版软件界面，标注其中的网格、层级、对比和图标抽象。
- 教学难度：入门—中级。
- 适合对象：本科生。
