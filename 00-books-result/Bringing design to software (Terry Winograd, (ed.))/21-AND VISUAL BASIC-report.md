## 二、文献定位

该章节以 HyperCard、Director 和 Visual Basic 为例，说明原型工具如何塑造交互设计的实践文化。它把 Schön 的“与材料对话”和 Schrage 的“原型文化”具体化到软件工具层面，展示不同编程 / authoring 环境如何影响设计者能做什么、如何迭代以及最终形成何种产品文化。

## 三、核心内容摘要

Winograd 指出，交互软件设计也存在与材料对话：程序员写代码、运行、体验、发现惊讶，再进入下一轮修改。快速原型工具的价值在于让设计者能迅速把未充分表达的想法变成可运行对象，并在使用中听到原型的反馈。不同工具对应不同原型目的和文化。

HyperCard 以卡片堆栈隐喻和 HyperTalk 语言面向普通 Macintosh 用户，支持 playful、简单、可探索的原型文化；Director 面向专业多媒体开发者，强调动画、声音、视觉效果和舞台 / 电影隐喻，适合高表现力但较复杂的原型；Visual Basic 面向 Windows 专业系统开发者，通过直接操纵界面构建器和 Basic 语言快速建立标准 Windows 应用，强调工程效率、执行性能和可交付性。章节最后指出，选择原型媒介会深刻影响设计过程，不能把工具看作中性的实现渠道。

## 六、教材使用建议

可用于“软件原型工具与数字设计文化”章节，说明 1990 年代交互设计形成时，工具并非中性容器，而是在普通用户、艺术设计师和专业程序员之间建立了不同的设计路径。

## 七、教学转化

课堂问题：同一个交互概念，用 Figma、代码、视频或纸面原型表达，会产生怎样不同的设计结果？作业可让学生比较三种原型工具的“文化”：它鼓励什么、限制什么、适合回答什么问题。
