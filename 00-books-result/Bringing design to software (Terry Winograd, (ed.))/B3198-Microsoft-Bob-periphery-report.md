## 二、原文实际包含什么

源文件实际是一篇关于 Microsoft Bob 的软件设计案例短文。文件开头有一条图片链接，随后以 Microsoft Bob 为对象，说明 Microsoft 在 1995 年初将 Bob 推出为面向家庭电脑新手的一类新软件。作者强调 Bob 并非以办公室用户为目标，而是面向可能害怕计算机、主要在家庭场景中使用软件的新手用户。

正文首先说明 Microsoft 试图让 Bob 显得不同于传统办公软件和既有操作环境。它不用常见的桌面隐喻，而以家庭房间的卡通场景作为界面：壁炉、家具、日历、支票簿和笔记本等对象对应日程、支付账单、写信等家用信息任务。文件随后指出，房间隐喻并非 Bob 首创，早期 HyperCard 应用、Magic Cap、Apple E-World 等也采用过房间、街道或城市隐喻。

文本接着分析 Bob 的卡通角色。Bob 不用抽象对话框作为主要互动方式，而通过用户可选择的动画角色与用户交流。作者把这种做法与 Clifford Nass 和 Byron Reeves 关于用户会把社会特征投射到计算机和传播技术上的研究联系起来，并举 Hopper 兔子和 Ruby 鹦鹉的提示语作为例子。

后半部分分析 Bob 如何通过“小而封闭的世界”、有限的应用和预包装选择，向新手传达“简单”和“有趣”。作者还把软件外部的名称、包装、Bob Magazine、广告语、广播广告和电视广告纳入设计对象，说明 Bob 的产品体验不只发生在运行程序时，也包括包装、产品名、推广和周边传播。结尾借 Brown 与 Duguid 对 Macintosh 包装的描述，强调软件体验的设计包括产品外围。

## 三、文献类型、作者立场与史料性质

本文不是 Microsoft 的官方说明，也不是用户研究报告，而是 *Bringing Design to Software* 中由 Terry Winograd 撰写的软件设计 profile。其史料性质属于二手设计评论与案例解读，包含对 Microsoft Bob 软件界面、推广方式和产品外围的观察，也嵌入作者对软件设计应关注用户体验外围的理论判断。

作者的立场是软件设计与 HCI 研究者、编者的分析立场。他关注的不是 Bob 的商业绩效，也不是完整产品史，而是 Bob 如何通过界面、角色、包装、广告和品牌语气建立一种“非办公室”“不困难”“有趣”的家庭计算语境。因此，文中有关 Microsoft 的设计意图应理解为作者对产品传播和界面策略的解读；有关 Bob 目标用户、广告语和包装内容的描述可以作为案例材料，但不能直接证明 Bob 实际改善了新手使用体验。

本文涉及企业产品与推广材料，必须区分四类内容：Microsoft 对 Bob 的定位与宣传，Bob 软件和包装中可观察到的设计策略，Winograd 对这些策略的解释，以及读者后续可能加入的市场成败判断。源文并未提供销量、用户满意度或长期使用效果数据。

## 五、论证结构与深度分析

本文的问题起点是：如果软件面向害怕计算机的家庭新手，设计如何在用户真正开始操作之前就降低心理门槛，并让产品被解释为不同于办公软件的新类型？作者的回答不是只看功能，而是把软件界面、交互语气和产品外围共同视为设计对象。

论证分为四层。第一层是目标用户与类型转换：Bob 被定位为家庭用户软件，而不是办公室软件。第二层是界面隐喻转换：Bob 用家庭房间取代桌面，把日历、支票簿、笔记本等日常物件放入房间场景，从而传达“这不是办公室”的意义。第三层是社交角色转换：Bob 通过动画角色与人格化语气与用户互动，试图让用户产生掌握感和连接感。第四层是外围传播：Bob 的名称、包装、Bob Magazine、广告语、广播和电视广告都被纳入统一的产品语境，强化简单、轻松、人人可用的形象。

文本最有价值的分析在于把“软件设计”从屏幕内扩展到屏幕外。作者明确指出，软件设计体验不始于也不止于运行程序，而包括包装、产品名称、促销传播和更大的产品世界。这一判断能帮助《现当代设计史》把软件纳入产品文化、品牌传播和用户经验史，而不是仅作为功能集合或代码系统来讨论。

需要注意的是，作者的论证并未证明 Bob 的策略有效。文中说明 Bob 是一个大规模、精心编排、有意识地设计产品外围的尝试，但没有提供用户研究、市场反馈或长期使用数据。因此，本文件更适合作为“外围设计如何被组织”的案例，而不能单独作为“Bob 成功改善新手计算体验”的证据。

本文件可用于《现当代设计史》中“数字技术与交互设计”“软件设计与用户经验”“消费软件与家庭计算”“品牌传播与产品外围设计”等章节。

具体使用方式包括：

- 作为软件界面隐喻案例，说明桌面隐喻之外还存在家庭房间、城市、角色代理等界面组织方式；
- 作为产品外围设计案例，说明包装、命名、说明物、广告和界面语气共同塑造软件类型；
- 作为家庭计算史材料，说明 1990 年代消费软件如何想象“电脑新手”和“家庭用户”；

## 七、可引用内容

### 1. 可直接引用的观点

- Bob 的目标用户被描述为家庭电脑用户，尤其是可能主要为家庭用途购买计算机和软件的新手。
- Bob 被分析为通过界面、推广和外围元素来制造一种不同于既有软件应用的新类型印象。
- 作者认为 Bob 的体验设计包括软件本身之外的名称、包装、促销和更大的产品世界。
- 文中明确提出，当谈论用户体验设计时，也是在谈论产品外围。

- Bob 使用卡通家庭房间作为主要界面场景，并用日历、支票簿、笔记本等对象进入具体任务。
