## 二、文献定位

该文是 Kees Dorst 对“设计思维”核心机制的系统阐释。它不是将设计思维简化为步骤工具，而是从推理模式、溯因、开放问题和框架创造解释设计思维为何能够被组织、商业、教育、医疗等领域借用。

## 三、核心内容摘要

Dorst 指出，设计思维在商业和管理领域流行之后，设计研究界必须回答“设计思维的核心是什么”和“它能为其他领域带来什么”。文章从形式逻辑出发，区分演绎、归纳和溯因，并进一步区分 Abduction-1 与 Abduction-2。设计的核心挑战在于开放性溯因，即在只知道希望创造的价值时，同时创造“what”和“how”。为应对这种双重未知，设计师发展出 framing 和 frame creation 的专业实践。文章最终指出，组织借用设计思维的价值不在于套用模糊工具箱，而在于学习框架创造，用新的问题理解方式开启组织创新。

## 四、研究对象

- 设计思维的核心机制；
- 溯因推理；
- frame / framing / frame creation；
- 组织问题解决；
- 设计实践向其他领域迁移。

## 六、核心概念

| 概念 | 英文 | 教材解释方向 |
|---|---|---|
| 溯因推理 | abduction | 从目标价值和不完整条件出发构造可能方案的创造性推理。 |
| 开放性溯因 | Abduction-2 | 设计中同时寻找“做什么”和“如何起作用”的双重未知推理。 |
| 框架 | frame | 以某种工作原则理解问题并通向价值创造的立场。 |
| 框架创造 | frame creation | 为复杂问题创造新的理解方式和行动路径。 |
| 设计思维 | design thinking | 专业设计实践中发展出的面对开放问题、创造价值和框架的能力。 |

## 七、论证路径

文章先说明设计思维流行带来的定义需求，再用逻辑模型解释演绎、归纳和溯因。随后，作者提出 Abduction-2 是设计推理核心，并转入 frame 的提出、测试和组织应用。文章最后讨论设计思维在组织创新中的可能性和误用风险。

## 九、教学转化建议

- 课堂问题：设计思维是流程图，还是一种推理能力？
- 讨论题：为什么复杂问题需要重新创造框架？
- 作业建议：选择一个组织问题，分别用既有框架和新框架提出两种方案。
- 教学难度：高级。
- 适合对象：本科高年级、研究生。
