# `00-books/` Markdown 深度报告生成与复审规则

## 版本：V1.1（目录说明与正式执行规则合并版）

- **生效日期**：2026-07-09
- **适用源目录**：`00-books/`
- **适用结果目录**：`00-books-result/`
- **路径基线**：仓库根目录 `books-paths.txt`
- **基准样例目录**：`00-books-result/[日]良品計畫株式會社：《MUJI 無印良品》，朱锷 译，2004/`

本文件由原 `00-books-result/README.md` 与 `00-books-result/DEEP-REPORT-GENERATION-AND-AUDIT-RULES.md` 合并而成，是 `00-books-result/` 的统一说明和正式执行依据。

本规则用于指导：

- 逐篇读取 `00-books/` 中的 Markdown 文档；
- 为缺失文档生成新的深度报告；
- 审查 `00-books-result/` 中已有深度报告；
- 纠正超出证据、内容虚构、类型误判和责任者混淆；
- 建立可用于《现当代设计史》研究、写作、教学和参考文献管理的可靠资料库。

深度报告的首要目标不是追求篇幅或数量，而是建立：

> 源文内容、文献性质、作者立场、历史证据、研究用途和不确定信息之间的清晰边界。

# 第一部分　目录与文件管理

## 一、目录目标

`00-books/` 中包含大量书籍全文、章节、导论、序言、目录、版权页、bibliography、索引、图版、OCR 拆分页和其他辅助文档。

`00-books-result/` 用于将每个源 Markdown 文档转化为可服务《现当代设计史》教材写作、论文研究、资料卡整理和课程教学的深度报告。

## 二、目录结构原则

`00-books-result/` 原则上采用与 `00-books/` 相同的相对目录结构。

源文件：

```text
00-books/某书名/某章节.md
```

对应报告：

```text
00-books-result/某书名/某章节-report.md
```

报告不应集中堆放在 `00-books-result/` 根目录，而应进入与原始文档对应的书籍或章节目录。

## 三、报告命名规则

常规命名：

```text
原文件名-report.md
```

示例：

```text
00-books/BUCHANAN, RICHARD. Branzi/01-Branzi's DilemmaDesign in Contemporary CultureRich.md
00-books-result/BUCHANAN, RICHARD. Branzi/01-Branzi's DilemmaDesign in Contemporary CultureRich-report.md
```

当文件名含特殊符号、题名过长或接口无法接受时，可以使用：

```text
B编号-简化题名-report.md
```

## 四、编号规则

报告内部保留 `B0001`、`B0002` 等管理编号。

1. 编号原则上参照 `books-paths.txt` 的路径顺序；
2. 已存在的编号不得随意重排；
3. 编号已被其他补建批次占用时，使用下一个未占用编号；
4. 报告编号不等同于真实完成数量；
5. 连续 OCR 页段可以合读，但必须列明全部源路径；
6. 重复文件不得重复计入可靠完成量；
7. 新编号与源路径的对应关系必须写入报告或批次记录。

# 第二部分　基本原则

## 一、源文件优先原则

每篇报告必须以对应的 `00-books/` Markdown 源文件为第一依据。

已有深度报告只能作为待审对象，不能作为事实来源。

不得：

- 依据文件名推测正文；
- 依据已有报告扩写已有报告；
- 依据同一本书其他章节补造当前文件内容；
- 依据常识或网络知识替原文补充论点；
- 把作者其他著作中的著名理论自动套入当前文件；
- 为满足篇幅要求添加原文没有的理论；
- 未完成源文复核便标记为“已完成”。

推荐执行顺序：

```text
读取源文
→ 判断文献类型与完整性
→ 检查前后文件、重复和串页
→ 读取并审查旧报告
→ 提取原文证据
→ 建立证据边界
→ 生成、修订或重写报告
→ 回读验证
```

## 二、信息量决定报告篇幅

不同文件不得机械套用同一篇幅。

### 1. 高信息量文件

包括：

- 完整章节；
- 长篇论文；
- 深度访谈；
- 理论文章；
- 企业发展评论；
- 设计案例分析；
- 带有多项史料和论证的正文。

要求形成完整深度报告，详细分析论证结构、人物、案例、概念、立场和史料性质。

### 2. 中等信息量文件

包括：

- 章节导言；
- 前言或序言；
- 部分正文；
- 人物简介；
- 作品说明；
- 课程史或机构说明。

应围绕文件实际功能进行分析，不得扩写为完整理论体系。

### 3. 低信息量文件

包括：

- 封面；
- 版权页；
- 目录；
- 缩略语表；
- 图版目录；
- 图片来源；
- 索引；
- 单一图注；
- OCR 残片；
- 空白页；
- 只有图片链接的文件。

只需说明：

- 文件实际包含什么；
- 可确认哪些元数据；
- 文件在整组文献中的作用；
- 哪些信息需要后续核验；
- 是否具有独立研究价值；
- 是否需要与相邻文件合读。

低信息量文件不得虚构理论价值。

## 三、文献类型先于理论分析

每篇报告必须首先判断文件属于什么文献类型。

可能类型包括：

- 学术专著正文；
- 论文；
- 前言或序言；
- 章节导言；
- 企业出版物；
- 企业宣传；
- 品牌评论；
- 机构院史；
- 周年纪念文本；
- 设计师访谈；
- 回忆录；
- 宣言；
- 广告；
- 产品说明；
- 图录；
- 人物简介；
- 作品清单；
- 目录；
- 版权页；
- 参考文献；
- 索引；
- OCR 残片；
- 空文件。

不同类型的材料具有不同证据价值。

不能把企业宣传、品牌纪念出版物、招生资料、机构院史或设计师回忆直接当作中立历史研究。

# 第三部分　深度报告固定结构

原则上，每份完整深度报告应包含以下十三部分。

低信息量文件可以适度合并章节，但不得省略证据边界、技术核验和问题诊断。

## 一、基础信息

至少包括：

- 报告编号；
- 源文件完整路径；
- 镜像报告路径；
- 文件标题；
- 所属著作或文献集；
- 作者、编者、译者或团体责任者；
- 出版时间或文件所涉年代；
- 文献类型；
- 文本完整性；
- 与相邻文件的关系；
- 复核状态；
- 建议等级。

建议等级：

| 等级 | 含义 |
|---|---|
| A 类 | 可作为《现当代设计史》核心理论或核心案例材料 |
| B 类 | 重要辅助材料、人物资料、版本资料或章节结构材料 |
| C 类 | 一般辅助材料、图注、目录入口或检索材料 |
| D 类 | 残缺文件、待合读文件或低信息量材料 |
| E 类 | 空文件、严重 OCR 损坏文件或暂不可用文件 |

## 二、原文实际包含什么

必须如实说明源文件中的实际内容，包括：

- 文章或章节的主要段落；
- 文件是否完整；
- 是否在句中截断；
- 是否混入多页内容；
- 是否包含图像链接；
- 是否包含目录、版权、图注或表格；
- 是否出现重复文本；
- 是否有明显 OCR 乱码；
- 是否与相邻文件连续。

本部分只回答：

> 原文件实际写了什么。

不得加入源文没有的解释。

## 三、文献类型、作者立场与史料性质

必须回答：

1. 谁在说；
2. 以什么身份说；
3. 为什么这样说；
4. 文本形成于什么语境；
5. 属于一手史料、二手研究还是混合材料；
6. 是否存在商业、机构、政治、纪念或职业立场。

例如：

- 企业管理者文章属于企业内部自述；
- 设计师访谈属于第一人称回忆；
- 周年出版物可能具有机构成功叙事；
- 品牌合作说明可能合理化商业项目；
- 设计评论可能具有支持品牌的倾向；
- 学术专著导论体现作者的方法和理论立场。

必须区分：

- 文本所陈述的事实；
- 作者的判断；
- 企业或机构的自我评价；
- 受访者的回忆；
- 报告撰写者的分析。

## 四、核心概念、观点、人物与案例

必须从原文中提取：

- 核心概念；
- 主要观点；
- 论证关系；
- 关键人物；
- 机构或企业；
- 作品、产品和项目；
- 技术与材料；
- 历史事件；
- 地点与年代；
- 数据或数量；
- 重要术语。

建议使用表格：

| 项目 | 原文中的含义 | 研究用途 | 证据状态 |
|---|---|---|---|
| 核心概念 | 当前文件中的具体解释 | 可用于哪类研究 | 已确认／待核 |
| 人物 | 身份和文本角色 | 人物史／机构史 | 已确认／待核 |
| 案例 | 原文实际描述 | 案例分析 | 已确认／待核 |

不得只列出词语而不解释其在当前文件中的具体含义。

## 五、论证结构与深度分析

高信息量正文必须进一步拆解论证结构，至少包括：

- 作者提出什么问题；
- 作者反对什么观点；
- 作者如何组织证据；
- 作者使用哪些案例；
- 结论如何从材料中产生；
- 存在哪些内部矛盾；
- 有哪些未处理的问题；
- 是否将商业利益、技术进步或设计价值自然化；
- 是否存在英雄史、目的论或欧洲中心论；
- 是否忽视女性、劳动者、供应商或非西方参与者。

分析必须依托原文，不得把外部理论强行套入。

## 六、对《现当代设计史》的具体用途

不得只写“具有设计史价值”。

必须明确文件可用于《现当代设计史》的哪个章节、知识点或研究问题。

可能章节包括：

- 工业革命与现代设计生产制度；
- 工艺美术运动与设计改革；
- 现代主义与功能主义；
- 包豪斯与现代设计教育；
- 战争、政治宣传与国家设计；
- 企业识别与品牌设计；
- 消费文化与广告；
- 女性、家庭劳动与性别化设计；
- 办公环境与科学管理；
- 交通、公共服务与导视设计；
- 日本现代与当代设计；
- 无品牌、日常生活与生活方式设计；
- 设计教育和专业制度；
- 新材料与制造技术；
- 数字技术与交互设计；
- 全球贸易、殖民体系与跨文化传播；
- 工作室工艺与当代工艺；
- 可持续设计与社会责任；
- 设计史方法、史料和写作问题。

还必须说明文件的具体使用方式：

- 理论依据；
- 历史案例；
- 设计师观点；
- 企业自述；
- 一手回忆；
- 图像材料；
- 反面案例；
- 机构宣传研究；
- 版本和责任者核验；
- 仅用于检索。

## 七、可引用内容

必须单独列出源文件中可以进入论文、教材或资料卡的内容。

可引用内容必须满足：

- 原文明确出现；
- 上下文相对完整；
- 责任者可确认；
- 不依赖无法读取的图片；
- 不属于明显 OCR 乱码；
- 不把企业自我评价改写成客观事实。

可以分为：

### 1. 可直接引用的观点

- 作者明确提出的定义；
- 作者明确的判断；
- 受访者明确的个人回忆；
- 企业文本明确提出的品牌原则。

### 2. 可作为案例的信息

- 作品名称；
- 设计者；
- 项目名称；
- 产品调整；
- 课程变化；
- 企业合作；
- 制造方式。

### 3. 可作为书目和人物资料的信息

- 版本；
- 出版者；
- 作者身份；
- 职务；
- 所负责章节。

所有正式引文仍应补核原书页码。

## 八、不能确认和需要外部核验的内容

必须明确列出不能确认的信息，包括：

- 缺少原书页码；
- 缺少作品年代；
- 缺少责任者；
- 图片无法读取；
- OCR 损坏；
- 译名存疑；
- 文件在句中截断；
- 企业自行公布的销量、排名或环保数据；
- 设计师多年后的回忆；
- 未提供来源的数字；
- 供应链、成本、劳动条件和环境绩效；
- 大型团队项目的个人署名问题；
- 机构对自身影响力的评价；
- 宣称“首次”“最大”“最成功”等内容。

建议使用明确措辞：

- 原文未提供；
- 当前文件不能确认；
- 需要回查原图；
- 需要与权威设计史资料核对；
- 需要与企业年报核对；
- 需要与原版版权页核对；
- 疑似 OCR 错误；
- 可能是翻译或上下文问题；
- 不得据此得出确定结论。

## 九、OCR、年份、版本和责任者问题

每份报告必须检查以下内容。

### 1. OCR 问题

- 错别字；
- 人名误识；
- 外文重音损坏；
- 数字粘连；
- 标点错乱；
- 图片文字混入；
- 页眉页脚混入正文；
- 两页或多页内容粘连；
- 单词断裂；
- 表格结构破坏。

### 2. 年份问题

- 文件名年份与版权页年份冲突；
- 原版年份和译本年份混淆；
- 初版与印次混淆；
- 作品年代和出版年代混淆；
- 回忆发生时间与文章写作时间混淆。

### 3. 版本问题

- 日文原版、英文版和中文版混淆；
- 重印和再版混淆；
- 简体版和繁体版混淆；
- 不同版本章节结构可能不同。

### 4. 责任者问题

- 作者与编者混淆；
- 译者与校译混淆；
- 设计者与制造商混淆；
- 摄影者与作品作者混淆；
- 企业与个人作者混淆；
- 团队项目被归给单个人；
- 学生作品与毕业后作品混淆；
- 历史设计者与当代再设计者混淆。

无法确认时不得擅自改正，只能标记为待核。

## 十、现有报告是否超出证据

复审已有报告时，必须逐项判断：

- 是否加入原文没有的人物；
- 是否加入原文没有的概念；
- 是否根据文件名推断主题；
- 是否把章节标题当成完整理论；
- 是否把后续文件内容提前写入；
- 是否根据图片链接虚构视觉分析；
- 是否把品牌宣言写成实际绩效；
- 是否把企业历史写成客观成功史；
- 是否把机构影响力写成已证实事实；
- 是否把设计师自述写成社会共识；
- 是否把可能性写成确定结论；
- 是否把目录、索引或版权页写出理论价值。

必须明确指出：

> 哪一句、哪一类判断，超出了什么证据。

## 十一、原报告问题诊断

现有报告的质量判断分为四级：

| 诊断 | 适用情况 |
|---|---|
| 可保留 | 与原文对应，证据边界清晰，仅需文字校订 |
| 需局部修订 | 核心内容正确，但缺少部分信息或存在个别越界 |
| 需大幅重写 | 结构过于简化，缺少立场、证据和使用边界 |
| 应废弃重建 | 与源文不对应、严重虚构、类型误判或事实错误 |

问题诊断应具体说明：

- 原报告正确之处；
- 原报告遗漏之处；
- 原报告错误之处；
- 哪些内容应删除；
- 哪些内容应降格为待核；
- 哪些部分需要与相邻文件合读。

## 十二、证据边界

报告结尾前必须明确说明：

- 本文件能够支持什么；
- 本文件不能支持什么；
- 哪些内容需要结合相邻文件；
- 哪些内容需要查阅图片；
- 哪些内容需要外部史料；
- 哪些内容只代表作者、企业或机构立场；
- 哪些数据具有时间性；
- 哪些理论只是章节入口，尚未展开。

证据边界是深度报告的必要组成部分，不得省略。

## 十三、审查结论

结论不能只是重复内容摘要。

应回答：

1. 本文件最可靠的研究价值是什么；
2. 是否可以进入《现当代设计史》正文；
3. 应以何种方式使用；
4. 是否需要继续合读；
5. 是否存在重大待核问题；
6. 当前报告是否达到可靠完成标准。

建议结论格式：

> 本文件可作为……章节的……材料。它能够可靠支持……，但不能单独证明……。正式引用前仍需核对……。现有报告已完成／尚未完成可靠复审。

---

# 第四部分　特殊类型文献的附加规则

## 一、企业、品牌和机构文本

凡涉及以下文献：

- 企业出版物；
- 品牌周年纪念册；
- 品牌合作说明；
- 企业领导者文章；
- 学校院史；
- 招生材料；
- 成功校友介绍；
- 企业产品说明；
- 环保宣传；
- 商业销售材料。

必须增加以下审查。

### 1. 叙述目的

判断文本是否用于：

- 建立品牌形象；
- 合理化企业扩张；
- 解释商业危机；
- 塑造创始神话；
- 建立机构权威；
- 宣传可持续形象；
- 证明设计成功；
- 推动具体商品销售。

### 2. 企业说法与外部事实分开

以下内容不得直接当作中立事实：

- 销售量；
- 市场排名；
- 产品环保性；
- 供应链公平；
- 劳动条件；
- 顾客真实体验；
- 社会影响；
- 国际化成功；
- 设计对生活的改善；
- 项目的“大众化”程度；
- 机构对行业的影响力。

应写为：

- 企业称；
- 作者认为；
- 文本将其描述为；
- 该项目被品牌解释为；
- 尚需企业年报或外部资料核验。

### 3. 英雄叙事检查

必须检查文本是否：

- 把团队成果归功于单一设计师；
- 把企业成功归功于创始人；
- 忽视工程师、制造商和供应商；
- 忽视劳动者和生产条件；
- 把市场成功等同设计价值；
- 把长期发展写成无挫折的连续胜利。

## 二、访谈、回忆和人物简介

### 1. 访谈和回忆

必须标明：

- 回忆可能发生在事件多年后；
- 记忆可能经过选择和重构；
- 个人经验不能代表所有学生、员工或使用者；
- 回忆可以作为一手经验，但不等于完整制度事实。

### 2. 人物简介

人物简介只能用于：

- 确认作者身份；
- 解释作者立场；
- 建立人物—章节关系；
- 提供成书时期的职业快照。

不得：

- 把人物其他著作中的理论自动套入当前文章；
- 把历史职位写成当前职位；
- 把奖项和机构介绍当作独立价值证明；
- 以人物简介替代正文分析。

## 三、图像、图注和图版文件

### 1. 图片可读取时

可以分析：

- 图片直接可见的布局；
- 色彩；
- 字体；
- 构图；
- 材料；
- 产品关系；
- 展示方式。

但必须区分：

- 图像直接可见信息；
- 图注提供的信息；
- 报告撰写者的解释。

### 2. 图片不可读取时

禁止分析：

- 色彩；
- 版式；
- 光线；
- 空间动线；
- 视觉风格；
- 图像象征；
- 产品使用状态。

只能记录：

- 图片链接存在；
- 图注文字；
- 已出现的作者、年代、作品名和馆藏信息；
- 后续需要补核原图。

### 3. 图版和正文关系

纯图版页不得当作完整理论章节。

必须判断：

- 是否承接前页；
- 是否属于正文案例；
- 是否为作品清单；
- 是否需要与前后文件合读。

# 第五部分　报告写作禁止事项

以下情况一律视为不合格：

1. 根据文件名写报告；
2. 只改标题和页码的模板化报告；
3. 所有文件使用相同分析段落；
4. 把封面或目录写成完整理论；
5. 根据不可读图片补写视觉细节；
6. 把企业宣传当作事实；
7. 把作者观点写成普遍共识；
8. 把设计师写成唯一作者；
9. 把市场成功等同设计价值；
10. 把环保外观等同环境绩效；
11. 把品牌低调等同无品牌；
12. 把“简洁”自动解释为功能主义；
13. 把残缺章节补写成完整理论；
14. 不检查 OCR、年份和版本；
15. 不检查作者、编者、译者和项目责任者；
16. 不写原报告问题诊断；
17. 不说明可引用和不可确认内容；
18. 只写抽象的“设计史意义”；
19. 为低信息量文件强行增加理论篇幅；
20. 使用未经原文支持的网络知识；
21. 未完成源文复核便标记为“已完成”；
22. 把未建方案、模型或概念写成已实施成果；
23. 把个人回忆写成完整制度事实；
24. 把图版题名误当作整个章节主题；
25. 忽略文件中的跨页、重复、截断或串页。

# 第六部分　质量验收标准

每份报告完成后必须进行自检。

## A. 源文对应

- 是否真实读取源文件；
- 是否说明文件实际内容；
- 是否判断完整性；
- 是否识别与前后文件的关系；
- 是否识别重复、串页和截断。

## B. 证据控制

- 每个主要判断是否有原文依据；
- 是否区分事实、作者观点和报告分析；
- 是否列出不能确认的信息；
- 是否存在图片或 OCR 越界；
- 是否把可能性改写成确定结论。

## C. 文献批判

- 是否判断文献类型；
- 是否说明作者身份；
- 是否分析企业、机构、政治或个人立场；
- 是否处理宣传和纪念属性；
- 是否识别英雄叙事和目的论。

## D. 学术用途

- 是否对应《现当代设计史》的具体章节；
- 是否说明材料应如何使用；
- 是否区分理论、案例、图像和辅助资料；
- 是否列出可引用内容；
- 是否说明正式引用所需的页码和版本核验。

## E. 技术核验

- 是否检查 OCR；
- 是否检查年份和版次；
- 是否检查责任者；
- 是否检查作品和项目署名；
- 是否区分原版、译本、重印和再版。

## F. 原报告复审

- 是否判断旧报告有无越界；
- 是否列出具体问题；
- 是否给出保留、局部修订、大幅重写或废弃结论；
- 是否说明删除、降格为待核或合读的内容。

# 第七部分　可靠完成的判定

只有同时满足以下条件，报告才可以计入“可靠完成量”：

1. 已读取对应源 Markdown；
2. 已判断文献类型和完整性；
3. 已说明作者、企业或机构立场；
4. 已提取实际概念、人物和案例；
5. 已明确《现当代设计史》的具体用途；
6. 已区分可引用和不可确认内容；
7. 已检查 OCR、年份、版本和责任者；
8. 已诊断原有报告；
9. 未对低信息量文件虚构理论；
10. 未将宣传性文本当作中立事实；
11. 已说明证据边界；
12. 报告内容不是批量替换式模板；
13. 已检查与相邻文件的连续、重复或串页关系；
14. 已在提交后回读验证报告文件。

不符合上述条件的报告只能标记为：

> 已生成，待二次复审。

不得计入最终可靠完成数量。

# 第八部分　建议执行状态

每份报告应在顶部或结尾明确标记以下一种或多种状态：

- `已完成可靠复审`；
- `需局部修订`；
- `需大幅重写`；
- `应废弃重建`；
- `待合读`；
- `待图像核验`；
- `待版本核验`；
- `待责任者核验`；
- `低信息量辅助文件`；
- `已生成，待二次复审`。
