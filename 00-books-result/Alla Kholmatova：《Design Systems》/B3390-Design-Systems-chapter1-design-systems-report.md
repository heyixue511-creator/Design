## 二、原文实际包含什么

文件定义 design system 为服务数字产品目的的互联 patterns 和 shared practices。它比较 Thomson Reuters Eikon 与 FutureLearn 的不同目的和模式，区分 functional patterns 与 perceptual patterns，引用 Christopher Alexander 的 pattern language，说明设计模式如何解决常见界面问题，讨论 shared language、pattern libraries 的局限，以及 effective design system 如何依赖 shared purpose。后部用 ten-minute recipe site 示例说明 purpose、values、design principles、behaviors/functional patterns、aesthetics/perceptual patterns 和 shared language 如何协同。

## 三、文献类型、作者立场与史料性质

这是全书核心定义章节。作者以实践者身份建立数字产品设计系统的工作模型。文本属于实践理论，不是行业标准文件；企业和历史案例多为说明性材料。

## 五、论证结构与深度分析

作者首先从定义切入，把设计系统建立在产品目的上，而不是建立在组件数量上。随后通过金融软件、在线学习、电商等例子说明不同领域产生不同功能模式，而品牌 ethos 产生感知模式。通过 Alexander，作者将界面模式和建筑模式语言相连，但并未把二者等同，而是借其说明共享语言和模式可复用性。对 pattern library 的批判是本章关键：即使是 living pattern library，也不能自动修复坏设计；如果系统本身僵化、沟通不足或目的不清，库只会暴露问题。

Ten-minute recipe site 示例将抽象概念落到设计过程：从目的和价值出发，再到原则、行为、功能模式、感知模式和共享语言。这说明作者的“系统”是目的驱动的，而不是工具驱动的。

## 六、对《现当代设计史》的具体用途

本文件可作为“数字技术与交互设计”中设计系统概念的核心材料，也可用于“设计管理与专业制度”“企业识别与品牌设计”。它帮助解释数字产品设计如何从单页面设计转向模式、库、语言和组织实践。
