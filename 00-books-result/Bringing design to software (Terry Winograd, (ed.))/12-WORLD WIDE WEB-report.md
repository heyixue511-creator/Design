## 二、文献定位

该章节以 Mosaic 和 World Wide Web 的早期扩散为案例，说明互联网成为大众媒介并非单纯因为技术突破，而是因为易用浏览器、极简协议、开放开发、分发网络和临界规模共同推动了社会性采纳。

## 三、核心内容摘要

Winograd 说明，1993 年互联网仍主要属于技术专家和“teenage hacker”式用户。1994 年 Mosaic 浏览器迅速扩散，使普通用户能够通过图形界面访问 World Wide Web。WWW 最初由 Tim Berners-Lee 及 CERN 同事为高能物理研究者沟通而开发，核心目标是让超文本在互联网中可用。Mosaic 则通过点击链接、显示文本与图像、免费分发和网络下载，降低了网络使用的门槛。

章节归纳了 Mosaic / WWW 的四个设计经验：临界规模现象、简单性作为关键设计原则、开放开发和分发网络的力量。WWW 的成功不是因为功能复杂，而是因为 HTML 与 HTTP 极其简单，允许大量开发者实验、扩展和快速迭代。开放标准和网络分发使浏览器在数天或数周内形成事实标准，这种分布式演化成为互联网设计的重要机制。

## 五、核心概念

| 概念 | 英文 | 教材解释方向 |
|---|---|---|
| World Wide Web | WWW | 基于超文本链接、HTML、HTTP 和浏览器的互联网信息系统。 |
| Mosaic | Mosaic | 早期图形化 Web 浏览器，使互联网进入大众视野。 |
| 临界规模 | critical mass | 足够多用户和内容参与后形成自我增强扩散的状态。 |

| 网络分发 | distribution networks | 通过互联网快速传播软件版本和事实标准的机制。 |

## 六、教材使用建议

可用于“Web 设计与互联网大众化”章节。教材中可把该案例与 Saffo 的消费者光谱相连：Mosaic 的价值在于降低使用复杂度，使网络从少数高阈值用户转向大众用户。

## 七、教学转化

课堂问题：为什么 Mosaic 的界面并不复杂，却能改变互联网历史？作业可让学生分析一个数字平台如何通过简单性、开放性和网络分发达到临界规模。
