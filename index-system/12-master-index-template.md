# 12 综合总表模板

如果只做一个综合索引表，建议至少保留以下字段。该表可作为 Excel、CSV、Notion、Obsidian Dataview 或数据库字段的基础。

## 一、综合总表字段

| 字段 | 说明 |
|---|---|
| 编号 | 文献编号，如 B001、P001 |
| 作者 | 作者姓名，外文作者保留原文 |
| 译者 | 翻译类文献填写 |
| 题名 | 书名或文章名 |
| 原书名 / 原文题名 | 外文著作或论文保留原题 |
| 文献类型 | 专著、论文、教材、译著、书评、报告等 |
| 出版信息 | 出版地、出版社、年份、期刊信息 |
| ISBN / DOI | 便于检索与核对 |
| 文件路径 | 仓库内 Markdown 文件路径 |
| 主题分类 | 属于哪个知识主题 |
| 关键词 | 3—8 个关键词 |
| 核心观点 | 文献最重要的观点 |
| 主要内容摘要 | 100—300 字摘要 |
| 适用章节 | 可用于教材哪一章 |
| 可用案例 | 文献中有哪些案例可用 |
| 可用概念 | 文献中有哪些概念可解释 |
| 可引用页码 | 重要页码或段落 |
| 教学用途 | 必读、选读、案例、拓展阅读、课堂讨论 |
| 重要性等级 | A / B / C / D |
| 版本与译本 | 原版、译本、所用版本、页码版本 |
| 图片资料 | 是否包含可用图片或图表 |
| 版权备注 | 图像或文本使用风险 |
| 待补 / 待核 | 信息缺失、需核对的问题 |
| 备注 | 其他说明 |

| 编号 | 作者 | 题名 | 类型 | 主题分类 | 关键词 | 核心观点 | 摘要 | 适用章节 | 教学用途 | 等级 | 文件路径 |
|---|---|---|---|---|---|---|---|---|---|---|---|
| B001 | Victor Margolin | World History of Design | 专著 | 全球设计史 | design history, global design, modernity | 设计史需要放入全球历史结构中理解。 | 本书以世界史视野组织设计发展，适合建立现代设计与全球历史之间的关系。 | 全球设计史章节 | 必读 / 框架文献 | A | `00-books/...` |
| P001 | Richard Buchanan | Wicked Problems in Design Thinking | 论文 | 设计研究 | wicked problems, design thinking | 设计面对的是复杂、开放且难以一次性解决的问题。 | 该文可用于说明设计思维与复杂问题的理论来源。 | 设计研究与设计思维 | 理论深化 / 课堂讨论 | A | `00-Paper/...` |

## 三、最终索引形态

一个成熟的《现当代设计史》教材文献索引，建议形成以下结构：

1. 总文献目录；
2. 按章节分类的文献索引；
3. 按主题分类的文献索引；
4. 核心概念索引；
5. 重要人物索引；
6. 经典案例索引；
7. 核心观点索引；
8. 可引用原文索引；
9. 图片与图表资料索引；
10. 教学设计索引；
11. 争议问题索引；
12. 待补与待核索引。

## 四、核心判断

用于撰写教材的文献、书籍索引，应具备三种功能：

1. **学术功能**：保证资料来源可靠、引用规范、观点有依据。
2. **写作功能**：帮助作者快速组织章节、提炼观点、安排案例。
3. **教学功能**：把学术资料转化为学生能够理解、讨论和学习的内容。

因此，它不应只是“参考文献清单”，而应是一个包含文献信息、主题分类、观点提炼、概念解释、案例材料、教学转化和引用依据的综合索引系统。
