## 三、核心内容摘要

导论指出，微处理器革命使计算机从专业机房进入日常工作与娱乐，软件产业也从硬件产业附属部分转变为重要经济部门。1990 年代软件产品形态发生变化，互动游戏、信息网络、智能饰品和个人通信器等新对象使“设计”成为软件领域的重要议题。

文本以 Kapor 的 Software Design Manifesto 为关键节点，说明软件行业开始反思可用性差和设计贫乏的问题。随后，ASD 的成立、ACM interactions 的创刊、CHI 会议中的设计议题、Quicken 的用户体验竞争优势、Mosaic / Netscape 推动互联网大众化等事件共同表明软件设计正在进入学科化阶段。

导论提出，软件设计应从 constructor's-eye view 转向 designer's-eye view。软件不只是程序结构或功能集合，而是创造 virtualities，即用户在其中感知、行动和回应的世界。软件设计类似建筑设计：它不仅指定结构，更塑造使用者生活其中的模式。

## 五、核心概念

| 概念 | 英文 | 教材解释方向 |
|---|---|---|
| 软件设计 | software design | 连接技术、用户、界面、经验和社会语境的软件创造实践。 |
| 构造者视角 | constructor's-eye view | 侧重内部结构、功能实现、可靠性与规格的工程视角。 |
| 设计者视角 | designer's-eye view | 以系统、用户、语境、价值和体验为整体起点的视角。 |
| 虚拟性 | virtuality | 软件生成的用户行动和意义空间。 |
| 软件中的居住 | inhabiting software | 将用户理解为生活在软件空间中的行动者。 |
