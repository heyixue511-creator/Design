## 三、已校正的重要误判

前一轮把许多“低信息量源文 + C 类辅助报告”算成 P0。抽查后确认：如果报告明确说明“仅作题名/封面/目录/索引识别”“无论证结构”“不宜作为正文证据”，这类报告可以保留为低信息量辅助报告，不应直接废弃。

例如：

- `B3416-Transforming-Technology-title-report.md`：源文仅 71 字，但报告明确标为 C 类题名页，不扩写理论，故不应算 P0。
- `B3429-Interior-Design-title-cover-report.md`：源文仅题名、作者、图片和系列名，报告明确限制为封面/题名信息，处理方向基本正确。
- `B3699-Title-Page-report.md`：源文是题名页，报告标为“待二次复审”，可进入低优先级复核，而非废弃重建。

## 四、第一批优先处理对象

### 1. 先补源路径再复审

这类报告最典型的问题是：报告正文可能已有分析，但没有建立 `00-books/` 源文件证据链。当前清单中有 1058 份。

优先处理策略：

1. 使用自动匹配的建议源文作为定位线索。
2. 人工打开源 Markdown，确认不是同名误配。
3. 回填源文件完整路径、文本完整性、相邻文件关系、证据边界。
4. 再判断是否可恢复为可靠完成量。

### 2. 源文为空/不可读却生成报告

这类目前有 173 份。处理原则：

1. 若源文确认为 0 字或只有 OCR 残片，报告标为 E/D 类或废弃重建。
2. 若是路径读取问题，先恢复可读性再复审。
3. 不得依据文件名、同组报告或常识补写理论。

### 3. 重复 B 编号

这类目前有 124 份。处理原则：

1. 同一 B 编号只能保留一个有效报告。
2. 重复报告需要判断是大小写冲突、旧版残留、同源重复，还是不同源误编号。
3. 重复项不得重复计入可靠完成量。
