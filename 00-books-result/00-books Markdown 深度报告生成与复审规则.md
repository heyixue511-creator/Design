# `00-books/` Markdown 深度报告生成与复审规则

## 版本：V2.0（语义分析与实证归类主导版）

- **生效日期**：2026-07-10
- **适用源目录**：`00-books/`
- **适用结果目录**：`00-books-result/`
- **路径基线**：仓库根目录 `books-paths.txt`
- **核心原则**：源文优先、证据边界、语义深读、知识要素解构、术语标准化、实证归类

本文件是 `00-books-result/` 的统一生成与复审规则。V2.0 取消旧版固定模板的机械要求，改用更适合章节 Markdown 的深度报告结构。报告不追求模板完整，而追求：

> 真实读取源文、准确判断文献性质、拆解知识要素、建立论证层级、归类实证材料、明确可用与不可用边界。

---

# 第一部分　适用对象与目录规则

## 一、处理对象

`00-books/` 中的每个切分 Markdown 原则上都应有对应深度报告。报告对象包括：

- 完整书籍或合并文件；
- 章节正文；
- 导论、前言、序言、后记；
- 访谈、宣言、论文、评论；
- 目录、版权页、索引、bibliography；
- 图版、图注、图片来源；
- OCR 残片、拆分页、重复页或串页。

用户明确要求“切分好的章节出示对应深度报告”时，应优先处理拆分后的章节文件。全书合并文件如与拆分章节重复，不应优先生成总报告，也不得与拆分章节重复计入可靠完成量。

## 二、目录与命名

报告应尽量进入与源文件对应的结果目录：

```text
00-books/某书名/某章节.md
00-books-result/某书名/某章节-report.md
```

若仓库已采用 `B0001-xxx-report.md` 形式，应继续沿用 B 编号，不强行改名。编号规则：

1. 已有编号不得重排；
2. 新报告使用下一个未占用编号；
3. 合读报告必须列明全部源路径；
4. 重复文件不得重复计入可靠完成量；
5. 新编号与源路径对应关系必须写入报告。

# 第二部分　基本原则

## 一、源文优先

每篇报告必须以对应 `00-books/` Markdown 源文件为第一依据。

不得：

- 依据文件名推测正文；
- 依据已有报告扩写已有报告；
- 依据同书其他章节补造当前文件内容；
- 依据常识或网络知识替原文补充论点；
- 把作者其他著作中的著名理论自动套入当前文件；
- 未完成源文复核便标记为“已完成可靠复审”。

推荐执行顺序：

```text
读取源文
→ 判断文件类型与完整性
→ 检查相邻文件、重复和串页
→ 提取知识要素
→ 标准化术语
→ 建立语义与论证层级
→ 归类案例、作品、史料和参考文献
→ 写报告
→ 回读验证
```

## 二、文献类型先于理论分析

必须先判断文件类型，再决定报告深度。常见类型包括：

- 学术专著正文；
- 论文或理论文章；
- 前言、序言、章节导言；
- 企业出版物、品牌文本、机构院史；
- 访谈、回忆录、人物简介；
- 宣言、广告、产品说明、图录；
- 目录、版权页、参考文献、索引；
- 图版、图注、图片来源；
- OCR 残片、空文件、重复文件。

不同类型具有不同证据价值。企业宣传、品牌纪念文本、机构自述和设计师回忆不得直接当作中立历史研究。

## 三、信息量决定篇幅

高信息量文件应形成完整深度报告；低信息量文件应如实降格，不得虚构理论价值。

| 信息量 | 典型文件 | 报告要求 |
|---|---|---|
| 高 | 完整章节、理论文章、深度访谈、案例分析 | 完整语义分析、知识解构、实证归类和教材用途 |
| 中 | 前言、导言、人物简介、作品说明、机构说明 | 围绕文件实际功能分析，不扩写成完整理论 |
| 低 | 封面、版权页、目录、索引、图注、空白页、OCR 残片 | 说明实际包含什么、元数据、待核事项和是否需合读 |

# 第三部分　V2.0 报告结构

V2.0 不再使用旧版固定结构。每篇报告应按以下结构组织。低信息量文件可以合并部分小节，但不得省略源文核验、证据边界和审查结论。

## 0. 报告头

必须包含：

- 报告编号；
- 执行状态；
- 建议等级；
- 图像状态；
- 源文件完整路径；
- 报告文件路径；
- 文件类型；
- 文本完整性；
- 相邻文件关系。

建议等级：

| 等级 | 含义 |
|---|---|
| A 类 | 核心理论、核心案例或关键史料 |
| B 类 | 重要辅助材料、版本资料、人物资料或章节结构材料 |
| C 类 | 一般辅助材料、图注、目录、索引或检索材料 |
| D 类 | 残缺、待合读、低信息量或严重 OCR 问题材料 |
| E 类 | 空文件、不可读文件或暂不可用材料 |

## 1. 源文核验与内容概述

本节只回答：源文件实际写了什么。

必须说明：

- 文件是否真实读取；
- 是否完整；
- 是否有句中截断、串页、重复或 OCR 损坏；
- 是否含图片链接、图注、表格、目录、版权页或 bibliography；
- 是否与前后文件连续；
- 文件实际段落或小标题结构。

不得在本节加入源文没有的解释。

## 2. 文献性质、作者立场与史料等级

必须回答：

1. 谁在说；
2. 以什么身份说；
3. 为什么这样说；
4. 属于一手史料、二手研究还是混合材料；
5. 是否存在企业、机构、商业、政治、纪念、教学或个人回忆立场；
6. 该文件在整本文献中的功能是什么。

必须区分：

- 原文陈述的事实；
- 作者判断；
- 企业或机构自我评价；
- 受访者回忆；
- 报告撰写者分析。

## 3. 语义深度分析

本节是高信息量报告的核心。必须回答：

- 文件的核心命题是什么；
- 作者如何定义或改变关键术语；
- 关键概念之间是并列、因果、递进、对立、转化还是例证关系；
- 哪些概念是描述性概念，哪些是规范性判断；
- 哪些概念只是章节入口，哪些已经展开为完整论证；
- 原文有哪些隐含前提；
- 原文是否把价值判断自然化为事实；
- 本章节与前后章节如何连接。

不得只写“作者认为……”。必须说明作者如何通过词语、例子、段落结构和证据建立判断。

## 4. 知识要素解构与术语标准化

必须扫描全文，拆解实际出现的知识要素。

### A. 知识要素表

| 要素类型 | 原文名称 | 标准英文／原文名 | 原文语境 | 可归入议题 | 证据状态 |
|---|---|---|---|---|---|
| 概念 |  |  |  |  | 已确认／待核 |
| 人物 |  |  |  |  | 已确认／待核 |
| 机构 |  |  |  |  | 已确认／待核 |
| 作品／项目 |  |  |  |  | 已确认／待图像核验 |
| 文献／史料 |  |  |  |  | 已确认／待版本核验 |

要素类型至少包括：

- 概念；
- 人物；
- 机构；
- 作品、产品、项目；
- 技术、材料、媒介；
- 历史事件、设计运动、展览；
- 文献、注释、bibliography、档案；
- 地理和时间信息。

### B. 术语标准化

核心概念和专有名词必须尽量补充标准英文对照或原文名。必须说明：

- 中文译名；
- 英文原名或外文原名；
- 常见缩写；
- 当前文件中的具体含义；
- 是否为作者自定义用法；
- 是否需要与通行译名核对。

不能确认时标为：

- `英文原名待核`；
- `译名待核`；
- `责任者待核`；
- `图像待核`；
- `版本待核`。

不得同一术语多译。不得擅自把未确认中文译名反推成英文术语。

## 5. 层级体系与论证结构

必须将源文知识组织成层级结构。高信息量章节至少包含三级：

```text
一级：章节核心问题
二级：主要论点／概念群
三级：案例、人物、作品、技术、史料或引文证据
```

建议表格：

| 层级 | 内容 | 原文证据 | 与《现当代设计史》的关系 |
|---|---|---|---|
| 一级问题 |  | 章节标题、导论段 | 对应教材章节 |
| 二级概念 |  | 小标题、关键段落 | 可形成知识点 |
| 三级证据 |  | 案例、图注、注释、bibliography | 可作为论据 |

低信息量文件可以简化，但必须说明为什么无法建立完整层级。

## 6. 实证归类：案例、作品、史料与参考文献

必须将全文出现的案例、作品、史料和参考文献分门别类归入议题，作为论据支撑。不得只罗列名称。

至少区分：

| 类别 | 包含内容 | 用途 |
|---|---|---|
| 设计作品／产品 | 产品、建筑、平面、服装、家具、界面、服务、系统 | 支撑案例分析 |
| 设计项目／展览 | 展览、研究项目、教学项目、公共项目 | 支撑制度与传播分析 |
| 人物与机构 | 设计师、作者、企业、学校、博物馆、出版社 | 支撑责任者与立场分析 |
| 技术与材料 | 工艺、媒介、材料、平台、算法、制造系统 | 支撑技术史分析 |
| 历史事件／运动 | 设计运动、社会事件、政策、战争、市场变化 | 支撑历史语境分析 |
| 文献与史料 | 书籍、论文、注释、bibliography、档案、图版来源 | 支撑引用和外部核验 |
| 图像与图注 | 图片链接、图版说明、摄影者、版权信息 | 支撑图像核验 |

建议表格：

| 议题 | 归入案例／作品／史料 | 原文证据位置 | 可支持的论点 | 待核事项 |
|---|---|---|---|---|
|  |  |  |  |  |

凡无法确认的案例不得强行归入确定议题；应标为“待核”“仅为图注线索”或“仅出现在 bibliography”。

## 7. 对《现当代设计史》的具体用途

不得只写“具有设计史价值”。必须明确：

- 可用于哪个章节；
- 可形成哪个知识点；
- 是理论依据、历史案例、设计师观点、企业自述、图像材料、反面案例、版本资料还是检索材料；
- 与哪些其他材料合读；
- 不能承担什么论证。

可能章节包括：

- 工业革命与现代设计生产制度；
- 工艺美术运动与设计改革；
- 现代主义与功能主义；
- 包豪斯与现代设计教育；
- 战争、政治宣传与国家设计；
- 企业识别与品牌设计；
- 消费文化与广告；
- 女性、家庭劳动与性别化设计；
- 交通、公共服务与导视设计；
- 日本现代与当代设计；
- 无品牌、日常生活与生活方式设计；
- 设计教育和专业制度；
- 新材料与制造技术；
- 数字技术与交互设计；
- 全球贸易、殖民体系与跨文化传播；
- 工作室工艺与当代工艺；
- 可持续设计与社会责任；
- 批判设计、推测设计与设计虚构；
- 设计史方法、史料和写作问题。

## 8. 可引用内容与不可确认内容

必须分别列出。

### A. 可引用内容

可引用内容必须满足：

- 原文明确出现；
- 上下文相对完整；
- 责任者可确认；
- 不依赖无法读取的图片；
- 不属于明显 OCR 乱码；
- 不把企业自我评价改写成客观事实。

可分为：

- 可直接引用的观点；
- 可作为案例的信息；
- 可作为书目和人物资料的信息；
- 可作为图像或项目线索的信息。

### B. 不可确认内容

必须列出：

- 缺少原书页码；
- 缺少作品年代；
- 缺少责任者；
- 图片无法读取；
- OCR 损坏；
- 译名存疑；
- 文件截断、串页或重复；
- 企业自行公布的销量、排名或环保数据；
- 设计师多年后的回忆；
- 未提供来源的数字；
- 供应链、成本、劳动条件和环境绩效；
- 机构对自身影响力的评价；
- 宣称“首次”“最大”“最成功”等内容。

建议措辞：

- 原文未提供；
- 当前文件不能确认；
- 需要回查原图；
- 需要与权威设计史资料核对；
- 需要与企业年报核对；
- 需要与原版版权页核对；
- 疑似 OCR 错误；
- 不得据此得出确定结论。

## 9. 技术核验与证据边界

必须检查：

- OCR 错别字、人名误识、数字粘连、标点错乱；
- 页眉页脚混入正文；
- 图片文字混入；
- 两页或多页内容粘连；
- 文件名年份与版权页年份冲突；
- 原版、译本、重印、再版混淆；
- 作者、编者、译者、摄影者、设计者、企业责任者混淆；
- 学生作品与毕业后作品混淆；
- 历史设计者与当代再设计者混淆。

证据边界必须说明：

- 本文件能够支持什么；
- 本文件不能支持什么；
- 哪些内容需要结合相邻文件；
- 哪些内容需要查阅图片；
- 哪些内容需要外部史料；
- 哪些内容只代表作者、企业或机构立场；
- 哪些数据具有时间性；
- 哪些理论只是章节入口，尚未展开。

## 10. 复审结论

结论不能只是摘要。必须回答：

1. 本文件最可靠的研究价值是什么；
2. 是否可以进入《现当代设计史》正文；
3. 应以何种方式使用；
4. 是否需要继续合读；
5. 是否存在重大待核问题；
6. 当前报告是否达到可靠完成标准。

建议格式：

> 本文件可作为……章节的……材料。它能够可靠支持……，但不能单独证明……。正式引用前仍需核对……。当前报告已完成可靠复审／已生成，待二次复审。

# 第四部分　特殊文件处理

## 一、全书合并文件

如果全书合并文件与拆分章节重复：

- 不优先生成总报告；
- 如需生成，只能作为目录级索引或合读辅助；
- 不得与拆分章节重复计入完成量；
- 用户要求章节报告时，应跳过合并文件，处理切分章节。

## 二、低信息量文件

封面、版权页、目录、索引、bibliography、图片来源、空白页、OCR 残片等低信息量文件只需说明：

- 文件实际包含什么；
- 可确认哪些元数据；
- 在整组文献中的作用；
- 是否需要合读；
- 哪些信息待核；
- 是否具有独立研究价值。

不得强行扩写理论意义。

## 三、图像、图注和图版文件

图片可读取时：

- 描述图像中可见对象；
- 区分作品、摄影、排版、图注；
- 记录图像与正文关系；
- 标记设计者、摄影者、版权机构。

图片不可读取时：

- 只能说明“文件含图片链接”；
- 不得补写视觉细节；
- 可根据图注提取作品名、作者、年份，但必须标为待图像核验。

## 四、企业、品牌和机构文本

必须区分：

- 企业事实；
- 企业自我叙述；
- 品牌价值宣言；
- 外部评论；
- 报告分析。

不得把企业宣传当作中立事实。不得把市场成功等同设计价值。不得把环保外观等同环境绩效。

## 五、访谈、回忆和人物简介

必须区分：

- 当事人回忆；
- 采访者提问；
- 后见之明；
- 传记作者判断；
- 可外部核验的事实。

不得把个人回忆写成完整制度事实。

# 第五部分　写作禁止事项

以下情况一律视为不合格：

1. 根据文件名写报告；
2. 只改标题和页码的模板化报告；
3. 所有文件使用相同分析段落；
4. 把封面或目录写成完整理论；
5. 根据不可读图片补写视觉细节；
6. 把企业宣传当作事实；
7. 把作者观点写成普遍共识；
8. 把设计师写成唯一作者；
9. 把市场成功等同设计价值；
10. 把环保外观等同环境绩效；
11. 把“简洁”自动解释为功能主义；
12. 把残缺章节补写成完整理论；
13. 不检查 OCR、年份和版本；
14. 不检查作者、编者、译者和项目责任者；
15. 不说明可引用和不可确认内容；
16. 只写抽象的“设计史意义”；
17. 为低信息量文件强行增加理论篇幅；
18. 使用未经原文支持的网络知识；
19. 未完成源文复核便标记为“已完成”；
20. 把未建方案、模型或概念写成已实施成果；
21. 把个人回忆写成完整制度事实；
22. 忽略跨页、重复、截断或串页；
23. 不做术语标准化；
24. 不做知识要素解构；
25. 不做案例、作品、史料和参考文献的实证归类。

---

# 第六部分　可靠完成判定

只有同时满足以下条件，报告才可以计入“可靠完成量”：

1. 已读取对应源 Markdown；
2. 已判断文献类型和完整性；
3. 已说明作者、企业或机构立场；
4. 已完成语义深度分析；
5. 已完成知识要素解构；
6. 已为核心概念和专有名词补充标准英文或原文对照；
7. 已建立层级体系；
8. 已将案例、作品、史料和参考文献按议题实证归类；
9. 已明确《现当代设计史》的具体用途；
10. 已区分可引用和不可确认内容；
11. 已检查 OCR、年份、版本和责任者；
12. 已说明证据边界；
13. 未对低信息量文件虚构理论；
14. 未将宣传性文本当作中立事实；
15. 报告内容不是批量替换式模板；
16. 已检查与相邻文件的连续、重复或串页关系；
17. 已在提交后回读验证报告文件。

不符合上述条件的报告只能标记为：

> 已生成，待二次复审。

不得计入最终可靠完成数量。

# 第七部分　建议执行状态

每份报告应明确标记以下一种或多种状态：

- `已完成可靠复审`；
- `已生成，待二次复审`；
- `需局部修订`；
- `需大幅重写`；
- `应废弃重建`；
- `待合读`；
- `待图像核验`；
- `待版本核验`；
- `待责任者核验`；
- `低信息量辅助文件`；
- `重复文件，不计入可靠完成量`。
