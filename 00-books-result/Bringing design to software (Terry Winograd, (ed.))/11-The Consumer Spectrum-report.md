## 二、文献定位

该章从消费者行为和高科技产品市场角度讨论软件与信息产品设计。它反驳“易用性就是成功关键”的简单判断，提出消费者是否接受产品取决于任务价值、忍耐阈值、支付能力、文化熟悉度和技术代际经验等多重因素。

## 三、核心内容摘要

Saffo 提出“threshold of indignation”（愤怒 / 忍耐阈值）概念，用于解释为什么某些复杂产品仍能成功，而某些友好产品却失败。人们使用工具不是因为工具友好，而是因为它能完成任务；当产品要求用户付出的努力超过可忍受界限时，用户才会放弃。商业办公用户、早期电脑爱好者、普通家庭消费者和高端消费者的阈值不同，因此产品设计不能只追求抽象的 user friendly。

章节进一步用“支付能力 / 愿意支付”和“忍耐阈值”两个维度建立消费者矩阵。高科技产品通常随着成本下降和性能提升向更低价格、更低忍耐要求区域移动；同时，用户也会随着文化熟悉度提高而提高阈值。这个模型解释了 DOS PC、VCR、电话、汽车、互联网等技术的市场扩散。

| V-B0017-05 | 企业常被其原有目标用户和价格结构锁定，难以转向新的消费区域。 | 管理型 | 设计战略 |

## 五、核心概念

| 概念 | 英文 | 教材解释方向 |
|---|---|---|
| 忍耐阈值 | threshold of indignation | 用户愿意为完成任务而忍受产品困难的最大限度。 |
| 消费者光谱 | consumer spectrum | 以支付能力和忍耐阈值区分高科技产品消费群体的模型。 |
| 用户友好 | user friendly | 产品易学、易用、少挫败的特征，但不是唯一成功因素。 |
| 任务价值 | task value | 用户为了完成某项任务而愿意承受的学习与使用成本。 |
| 技术熟悉度 | technological familiarity | 用户因文化经验、代际环境和长期使用形成的技术适应能力。 |

## 六、教材使用建议

可用于“数字产品的用户与市场”章节，说明 UX 不能只被讲成界面易用性，还要考虑任务价值、消费者类型、产业结构和技术文化熟悉度。它也能与信息家电、Web 普及和服务设计章节建立联系。

## 七、教学转化

课堂问题：为什么有些难用的软件仍然成功，有些好用产品却失败？作业可让学生把手机、智能家居、VR、AI 工具等放入 Saffo 的消费者矩阵中，解释其采纳难点。
